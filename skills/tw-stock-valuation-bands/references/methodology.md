# 估值規則

## 用途

這個 skill 把台股財報分析 JSON 轉成：

- 悲觀 / 中性 / 樂觀三種 EPS 情境
- 每個情境對應的 `1–5 分` 價格區間
- 指定現價時的 PER 與評分
- 若提供歷史股價 JSON，補上近年收盤價區間、區間位置、收盤價百分位

## 預設規則

- 基準 EPS：採 `metrics_by_year` 最新年度的 `eps`
- 悲觀 EPS：`base_eps * (1 - 15%)`
- 中性 EPS：`base_eps`
- 樂觀 EPS：`base_eps * (1 + 15%)`

需要更保守或更樂觀時，可改用腳本參數覆寫：

- `--pessimistic-eps`
- `--base-eps`
- `--optimistic-eps`
- `--scenario-downside`
- `--scenario-upside`

## 1–5 分對應

| 分數 | PER 區間 | 解讀 |
|---|---|---|
| 5 | `< 8x` | 明顯便宜 |
| 4 | `8x–9x` | 偏便宜 |
| 3 | `9x–10x` | 合理 |
| 2 | `10x–11x` | 偏貴 |
| 1 | `>= 11x` | 昂貴 |

## 使用時機

- 使用者要把財報分析接到股價
- 使用者要估 `合理價`
- 使用者要三情境價格帶
- 使用者要 `1–5 分` 估值評分
- 使用者要分批布局區間

## 限制

- 這套規則是簡化估值帶，不是完整 DCF
- 對循環股特別要搭配景氣判讀
- 若最新年度 EPS 含一次性利益，應先用自訂 EPS 覆寫
- 歷史股價支援上市 TWSE 與上櫃 TPEx 公開日價 JSON（`fetch_price_history.py --market twse｜tpex`）
- 區間位置與百分位是價格相對位置，不等於完整歷史 PER 回測

## 歷史股價補充

若估值報告另外帶入 `price_history_json`：

- 區間低點：該期間所有收盤價最小值
- 區間高點：該期間所有收盤價最大值
- 區間位置：`(最近收盤 - 低點) / (高點 - 低點)`
- 百分位：以期間內所有收盤價排序後，最近收盤落點的累積百分比

這組資料適合回答：

- 現價相對近年股價是低檔、中段、還是高檔
- 現在的價格是不是已經接近近年高點

不適合直接當成：

- 完整歷史本益比分位數
- 已處理除權息的還原價回測
